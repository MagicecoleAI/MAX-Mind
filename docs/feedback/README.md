## 목적

**"왜 그렇게 결정했는가"의 이력을 남긴다.**
대화·PR 코멘트에 흩어지는 피드백을 단일 위치에 수집해, 3개월 후 같은 논쟁을 반복하지 않도록.

**이 디렉토리가 해결하는 문제**:
- "이거 왜 이렇게 했지?"에 답하지 못하는 코드
- 코드 리뷰에서 반복되는 같은 지적
- 지금 해결 못하지만 잊으면 안 되는 이슈 (백로그)

---

## 파일 구성

| 파일/디렉토리 | 용도 |
|------|------|
| [backlog.md](backlog.md) | 지금 해결 못하는/당장 안 할 항목 누적 (`B-NNN`) |
| `reviews/YYYY-MM-DD-T-NNN.md` | 타스크별 코드 리뷰 기록 (선택) |
| `decisions/YYYY-MM-DD-slug.md` | 중요 의사결정 기록 (ADR-lite) |

> **최소 시작**: `backlog.md` 하나만 먼저 써도 된다. reviews/decisions/는 필요해지면 추가.

## 피드백 4유형 분류

피드백을 받았을 때 어떤 유형인지 먼저 판단하고 대응한다.

| 유형 | 뜻 | 즉시 행동 |
|------|-----|---------|
| 🔄 **즉시 수정** | 방향이 틀렸음. 원복 필요 | 논쟁 전에 먼저 `git checkout` 또는 revert |
| ↪️ **방향 전환** | 접근 방식 자체 재검토 | 현재 작업 중단, plan 섹션 2 수정 |
| ℹ️ **정보 제공** | 정확한 수치/맥락 보충 | 진행 로그에 기입, 계속 진행 |
| 📋 **백로그 등록** | 지금 해결 불가. 별도 추적 | [backlog.md](backlog.md)에 등록 |

### 판단 가이드

- "이거 왜 이렇게 했어?" → **정보 제공** (설명 후 진행)
- "이거 엉뚱한 걸 고쳤네" → **즉시 수정** (원복)
- "접근 자체가 문제 같은데" → **방향 전환** (plan 재작성)
- "나중에 할까?" → **백로그 등록** (backlog.md)

## 의사결정 기록 (ADR-lite) 템플릿

`decisions/2026-04-18-xmind-format-preserve.md`

```markdown
# ADR: XMind 포맷 식별자 보존 결정

> **최종 수정**: 2026-04-18 · **담당**: @magic · **상태**: 🟢 확정

## 컨텍스트
프로젝트 이름을 `xmind` → `MAX Mind`로 변경했으나, 코드 전역에
`XMindSheet`, `XMindTopic`, `_xmindRaw`, `.xmind` 확장자, `"XMind Files"` 다이얼로그 필터 등
XMind 식별자가 다수 존재.

## 결정
**XMind 식별자는 보존한다.** 프로젝트 이름 변경과 무관하게 유지.

## 고려한 대안
1. **전부 `MaxMind...`로 리네임**: 파일 호환성 설명 부담 커지고, 외부 XMind 도구와의 왕복 호환성 검증 비용 폭증
2. **중간 어댑터 계층**: 코드 복잡도 증가, 득 없음
3. **보존** ← 선택

## 근거
- XMind Zen은 업계 표준 포맷. 타입명이 스펙 용어와 일치할 때 호환성 디버깅이 쉽다
- [code-quality.md:483] "XMind 파일 호환은 정상 동작, 건드릴 이유 없음" 기존 방침과 정합
- 사용자 식별 이름(`MAX Mind`)은 UI·문서에서 관리하고, 내부 포맷 식별자는 스펙 용어 유지

## 영향
- 코드 내 `XMind*` 명칭은 프로젝트 이름이 아니라 파일 포맷을 가리킨다는 점을 CLAUDE.md에 명시
- 신규 합류자의 혼동 방지를 위해 `src/services/xmindConverter.ts` 상단에 주석 추가 검토

## 참조
- [../changelog.md](../changelog.md) "2026-04-18 · MAX Mind 리브랜딩"
- [../../CLAUDE.md](../../CLAUDE.md) §6 (❌ 하지 않기)
```

## 작성 규칙

1. **피드백 받은 즉시 유형 판단** → 적절한 위치에 기록
2. **의사결정은 가능하면 당일 기록**. 일주일 지나면 근거가 흐릿해진다
3. **backlog.md는 주기적으로 정리** (월 1회 등) — 쌓이기만 하면 소용없다
4. **폐기된 결정도 남긴다** — 왜 폐기했는지 추가 기록 (같은 제안 재림 방지)
