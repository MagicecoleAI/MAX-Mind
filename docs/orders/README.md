## 목적

**"오늘 우리는 뭘 하고 있는가?"** 를 한눈에 보는 단일 진실 원천.
세션 시작 시(팀원·AI 에이전트 모두) 맥락을 즉시 로드할 수 있게 한다.

**이 디렉토리가 해결하는 문제**:
- 대화/메모에 흩어진 "오늘 뭐 하시는지" 질문
- PR만으로는 보이지 않는 **진행 중/블로커 상태**
- Claude Code 세션 시작 시 매번 재설명해야 하는 현재 맥락

---

## 파일 포맷

### 옵션 A — 일일 (daily)
- 경로: `docs/orders/YYYY-MM-DD.md` (예: `2026-04-18.md`)
- 적합: 변화가 많은 주, 여러 사람이 동시 작업 중
- 부담: 매일 작성 (3~5분)

### 옵션 B — 주간 (weekly)
- 경로: `docs/orders/WYYYY-NN.md` (예: `W2026-16.md` = 2026년 16주차)
- 적합: 소규모 팀(2~5명), 장기 집중 작업
- 부담: 주 1회 + 중간 업데이트

> **MAX Mind 현재 운영**: 1인 개발 단계 → **일일(A) 방식**을 기본으로, 필요 시 주간으로 전환.

## 작성 규칙

1. **타스크는 반드시 `T-NNN` 형식으로** 참조. plans/ 파일과 1:1 매칭.
2. **담당자는 항상 `@handle` 명시**. 주인 없는 타스크는 진전이 없다.
3. **블로커는 무엇을 기다리는지 구체적으로**. "대기 중" ❌ → "XMind 샘플 파일 검토 대기" ✅
4. **완료된 것은 과감히 삭제하지 말고 ✅로 표시**. 히스토리 가치가 있다.
5. **다음 날/주 파일에는 이전 파일의 미완료만 이월**. 전체 복붙 금지.

## Claude Code와의 협업

AI 에이전트 세션 시작 시 이 디렉토리의 최신 파일을 먼저 읽게 하면 효율이 급등한다.

**대화 시작 예시**:
> "오늘 작업 시작 전에 `docs/orders/`의 가장 최신 파일 읽고, 내가 이어가야 할 타스크를 알려줘."

---

## 누적된 파일 관리

- **삭제하지 않음** (히스토리 자산). 깃 기록과 함께 쌓임.
- **3개월 이상 지난 파일**은 `docs/orders/archive/YYYY/` 하위로 이동 권장 (선택).
- **인덱스가 필요할 만큼 쌓이면** 월별 요약 파일 생성 고려 (`2026-04-summary.md`).
