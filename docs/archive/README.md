## 목적

더 이상 최신이 아닌 문서를 **삭제하지 않고 보관**하여 맥락·근거를 보존한다.

**원칙**: 삭제 대신 이동(Move over Delete).

---

## 이동 기준

다음 중 하나에 해당하면 이 디렉토리로 이동 고려:

1. **대체됨**: 같은 주제의 최신 문서가 별도 경로에 존재하며, 구 문서는 참조용으로만 의미 있음
2. **폐기됨**: 더 이상 유효하지 않은 전략·계획
3. **오래됨**: 3~6개월 이상 갱신 없고 현재 프로젝트 상태와 괴리가 명확함
4. **주간/일일 orders**: 3개월 이상 지난 `orders/` 파일 (선택)

---

## 이동 절차

1. 이동 전 원본 위치를 확인 (`git log --oneline -- path/to/doc.md`)
2. `git mv docs/some/path.md docs/archive/path.md` (이력 보존)
3. 이동된 문서 상단에 아카이브 헤더 추가:
   ```markdown
   > **아카이브됨**: 2026-XX-XX · **사유**: 대체 문서 [../new-doc.md](../new-doc.md)로 일원화
   > **원본 경로**: docs/old/path.md
   ```
4. 상태 이모지를 🔴로 변경
5. [INDEX.md](../INDEX.md)의 해당 엔트리 제거 또는 아카이브 섹션으로 이동
6. `changelog.md`에 1줄 기록

## 디렉토리 구조 (쌓일수록 세분화)

```
archive/
├── README.md                       (이 파일)
├── (초기에는 평탄)
│
└── (쌓이면 연도별 분할)
    ├── 2026/
    │   ├── docs/                   (이동된 설계/가이드)
    │   ├── orders/                 (오래된 작업 지시서)
    │   └── plans/                  (폐기된 타스크)
    └── 2027/
```
